# US-002 — Suggest reports no match on an unrelated task (true negative)

**As an** agent working on a task unlike anything in the library
**I want** `chant suggest` to return no candidate rather than a low-quality guess
**So that** I am not nudged into reusing the wrong recipe (no wrong-answer amplification).

## Acceptance criteria

- [ ] Running `chant suggest --task "rotate the kubernetes TLS certificates in the staging cluster" --json` against the demo library **must** exit `0`.
- [ ] The outcome **must** set `"match_found": false` and **must** omit or empty the `hits` array.
- [ ] The outcome `recommended_next_command` **must** instruct the caller to solve and capture (the literal `no recipe matched — solve the task, then \`chant capture\` it`).
- [ ] In human (non-`--json`) mode the output **must** state `no recipe matched "<task>" above threshold <T>` where `<T>` is `retrieval.threshold` (default `0.25`).
- [ ] A recipe whose blended score is below `retrieval.threshold` **must not** appear in `suggest` output (it may still appear in `search`).
