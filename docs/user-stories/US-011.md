# US-011 — Search ranks recipes against a free-text query

**As a** human or agent exploring the library
**I want** `chant search "<query>"` to rank all recipes by relevance
**So that** I can find candidates even below the suggest threshold.

## Acceptance criteria

- [ ] Running `chant search "revenue by channel" --json` **must** exit `0` and emit `"subcommand": "search"` with a `hits` array.
- [ ] Unlike `suggest`, `search` **must** rank and return all recipes (it does not apply `retrieval.threshold`).
- [ ] In human mode each row **must** show the blended score and its components: `(lex <L>, signal <S>, ok <R>)`.
- [ ] Hits **must** be sorted by descending blended score, ties broken by ascending `id` (deterministic ordering).
- [ ] `chant search` with no query **must** fail (exit `1`) with `search requires a query`.
- [ ] On an empty library `chant search "<q>"` **must** print `no recipes to search.`
