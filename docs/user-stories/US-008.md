# US-008 — Invalidate marks a recipe stale

**As a** library maintainer who noticed a recipe stopped working
**I want** `chant invalidate` to flag the recipe stale
**So that** retrieval de-prioritizes it until it is re-verified.

## Acceptance criteria

- [ ] Running `chant invalidate --reason "schema changed" csv-revenue-by-channel --json` **must** exit `0` and set the recipe's `status: stale` on disk.
- [ ] The outcome **must** set `"subcommand": "invalidate"`, `"stale": true`, and a `message` noting a passing `chant verify` will re-bless it (the reason **should** be appended in parentheses).
- [ ] After invalidation, `chant list` **must** flag the recipe `(stale)` and `chant status` **must** count it under `stale_count`.
- [ ] A stale recipe **must** still be retrievable by `chant suggest` / `chant search` but its blended retrieval score **must** be halved, and its hit `reasons` **should** include `recipe is stale — re-verify before trusting`.
- [ ] `chant invalidate` with no id **must** fail (exit `1`) with `invalidate requires a recipe id`.
- [ ] NOTE: `--reason` / `--json` **must** precede the recipe id on the command line.
