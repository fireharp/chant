# US-016 — Init scaffolds a chant-enabled repo

**As a** developer adding chant to a repo
**I want** `chant init` to scaffold config, recipes dir, the agent skill, and gitignore
**So that** I can start capturing enchantments immediately.

## Acceptance criteria

- [ ] Running `chant init --json` **must** exit `0` and emit `{ "created": [...], "skipped": [...], "root": "<path>" }`.
- [ ] Init **must** create `chant.yml`, `recipes/.gitkeep`, and `.agents/skills/chant/SKILL.md` when absent.
- [ ] Init **must** ensure `.chant/` is gitignored (appending to `.gitignore`, not clobbering it) and create the `.chant/` state dir.
- [ ] Re-running `chant init` without `--force` **must** list already-present files under `skipped` and not overwrite them.
- [ ] Running `chant init --force` **must** overwrite the scaffolded files and list them under `created`.
- [ ] After `chant init`, `chant doctor` **must** report `config` `ok`, `recipes-dir` `ok`, and `gitignore` `ok`.
