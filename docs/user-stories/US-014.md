# US-014 — Status writes a human-readable library snapshot

**As a** human reviewing the recipe library's health
**I want** `chant status` to write and report a STATUS.md snapshot
**So that** I can see counts, trust coverage, and per-recipe metrics.

## Acceptance criteria

- [ ] Running `chant status --json` **must** exit `0` and emit a report with `generated_at`, `recipe_count`, `active_count`, `stale_count`, `total_runs`, `total_failures`, and a `recipes` array.
- [ ] Each `recipes[]` entry **must** carry `id`, `version`, `status`, `runs`, `failures`, `success_rate`, and `has_verifier`.
- [ ] `chant status` **must** write `.chant/STATUS.md` with a table and a verifier-first reminder note.
- [ ] In human mode the output **must** state `wrote <path>/.chant/STATUS.md (<N> recipe(s), <A> active, <S> stale)`.
- [ ] `active_count + stale_count` **must** equal `recipe_count`.
