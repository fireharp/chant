# US-004 — Capture without a verifier warns that reuse cannot be trusted

**As a** library maintainer
**I want** `chant capture` to flag a recipe that has no verifier
**So that** I do not accumulate recipes whose reuse can never be trusted.

## Acceptance criteria

- [ ] Running `chant capture --id no-verifier --task "x" --command "echo hi"` (no `--verifier`, no `--expect-artifacts`) **must** still exit `0` and create the recipe.
- [ ] In human mode the output **must** include a warning that no verifier is set and that a hit without a verifier is just a guess.
- [ ] A later `chant doctor` **must** report the `verifiers` check as `warn` with a count of recipes that have no verifier (e.g. `N/M recipe(s) have no verifier — reuse can't be trusted`).
- [ ] A later `chant verify no-verifier` **must** fail (exit `1`) because there is no verifier command or expected artifacts to establish trust.
- [ ] `chant suggest` **may** still return this recipe as a candidate, but its hit **must** report `"verifier_exists": false`.
