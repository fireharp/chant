# US-010 — List browses the recipe library

**As a** human or agent surveying what is cached
**I want** `chant list` to enumerate recipes with their track record
**So that** I can see what is available at a glance.

## Acceptance criteria

- [ ] Running `chant list --json` **must** exit `0` and emit the index object with `generated_at`, `count`, and a `recipes` array.
- [ ] Each `recipes[]` entry **must** carry `id`, `version`, `description`, `status`, `runs`, `failures`, and `success_rate`.
- [ ] `chant list` **must** rebuild `.chant/index.json` as a side effect (the index reflects current on-disk recipes).
- [ ] In human mode each row **must** show `<id> v<version>  <runs> run(s) <pct>% ok` and append ` (stale)` when `status == "stale"`.
- [ ] On an empty library `chant list` **must** print `no recipes yet — capture one with \`chant capture\`.` and `count` **must** be `0`.
