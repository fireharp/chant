# US-012 — Explain prints a recipe card

**As a** human or agent deciding whether to reuse a recipe
**I want** `chant explain <id>` to print the full recipe card
**So that** I can read its applicability, procedure, verifier, and metrics.

## Acceptance criteria

- [ ] Running `chant explain csv-revenue-by-channel` **must** exit `0` and print the id, version, status, description, task patterns, tags, the `what to do` command, and the verifier command.
- [ ] When a recipe has no verifier, `chant explain` **must** print `⚠ no verifier — reuse cannot be trusted.`
- [ ] `chant explain` **must** print a metrics line: `<runs> run(s), <failures> failure(s), <pct>% success`.
- [ ] `chant explain <missing-id>` **must** fail (exit `1`) with `recipe "<id>" not found`.
- [ ] `chant explain` with no id **must** fail (exit `1`) with `explain requires a recipe id`.
- [ ] GAP NOTE: `chant explain --json <id>` emits the raw recipe struct with Go field names (e.g. `ID`, `Version`, `WhatToDo`), NOT the snake_case outcome contract used by the other subcommands. Flagged for the lead.
