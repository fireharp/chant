# US-003 — Capture distills solved work into a versioned recipe

**As an** agent that just solved a recurring task and passed a verifier
**I want** `chant capture` to persist the procedure as a recipe card
**So that** the next similar task can reuse it.

## Acceptance criteria

- [ ] Running `chant capture --id my-recipe --task "do the thing" --command "python3 run.py {{input}}" --verifier "python3 test_recipe.py" --json` **must** exit `0` and write `recipes/my-recipe/recipe.yaml`.
- [ ] The outcome **must** set `"subcommand": "capture"`, `"captured": true`, `"recipe_id": "my-recipe"`, and `"version": 1`.
- [ ] The outcome `recommended_next_command` **must** be `chant verify my-recipe` (capture establishes no trust on its own).
- [ ] When `--id` is omitted, the recipe id **must** be derived as a slug of `--task` (lowercase, non-alphanumerics collapsed to single dashes, trimmed).
- [ ] Capture **must** fail (exit `1`) when neither `--task` nor `--id` is given, with message `capture requires --task or --id`.
- [ ] Capture **must** fail (exit `1`) when `--command` is missing, with message `capture requires --command (the procedure to reuse)`.
- [ ] Re-capturing an existing id without `--force` **must** fail (exit `1`) with a message advising `--force` or invalidate + recapture.
- [ ] The captured card **must** record `metrics.runs: 0` and `metrics.failures: 0` (no track record until verified).
