# US-013 — Index rebuilds the retrieval index

**As a** maintainer or CI step
**I want** `chant index` to rebuild `.chant/index.json` from the recipe library
**So that** listing and tooling reflect the current on-disk recipes.

## Acceptance criteria

- [ ] Running `chant index --json` **must** exit `0` and emit the index object with `generated_at`, `count`, and `recipes`.
- [ ] `chant index` **must** write `.chant/index.json` under the repo root's gitignored state dir.
- [ ] In human mode the output **must** state `indexed <N> recipe(s) → <path>/.chant/index.json`.
- [ ] The index `count` **must** equal the number of directories under `recipes_dir` that contain a `recipe.yaml`.
- [ ] `chant index` **must** succeed (count `0`) on an empty library without error.
