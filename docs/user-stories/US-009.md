# US-009 — Re-verifying a stale recipe re-blesses it to active

**As a** library maintainer who repaired a stale recipe
**I want** a passing `chant verify` to flip its status back to active
**So that** the invalidate -> repair -> re-verify loop restores trust.

## Acceptance criteria

- [ ] Given a recipe with `status: stale`, running `chant verify --json <id>` **must**, on a verifier pass, set `"trusted": true`.
- [ ] On that passing verify the recipe's on-disk `status` **must** change from `stale` back to `active`.
- [ ] After re-blessing, `chant suggest` for a matching task **must** no longer apply the stale score penalty and the hit `reasons` **must not** include the stale warning.
- [ ] If the verifier still fails, the recipe **must** remain `stale` and the outcome **must** keep `"trusted": false`.
- [ ] `chant status` **must** reflect the recipe moving from `stale_count` back to `active_count` after a successful re-verify.
