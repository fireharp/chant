# US-005 — Run executes a recipe but never establishes trust

**As an** agent reusing a recipe
**I want** `chant run` to execute the procedure with adapted inputs
**So that** I produce output — while understanding the result is not yet trusted.

## Acceptance criteria

- [ ] Running `chant run --input input=examples/orders_shopify.csv csv-revenue-by-channel --json` **must** substitute `{{input}}` and execute `python3 run.py examples/orders_shopify.csv` in the recipe directory.
- [ ] The outcome **must** set `"subcommand": "run"`, `"executed": true`, an `exit_code`, and `runtime_ms`.
- [ ] The outcome **must** set `"trusted": false` always — `run` alone never establishes trust.
- [ ] The outcome `recommended_next_command` **must** be `chant verify <id>`.
- [ ] When the executed command exits non-zero, `chant run` **must** exit `1` and surface the command's stderr.
- [ ] Inputs **must** be exposed to the procedure both via `{{var}}` substitution and as `CHANT_INPUT_<KEY>` environment variables.
- [ ] NOTE: `--input` and `--json` are flags and **must** precede the recipe id on the command line (Go flag parsing stops at the first positional arg).
