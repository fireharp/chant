# US-017 — Bench runs the validation suite and gates on failure

**As a** maintainer or CI pipeline
**I want** `chant bench` to run the retrieval and e2e suites and fail on any scenario failure
**So that** the verifier-first thesis stays proven on every change.

## Acceptance criteria

- [ ] Running `chant bench --json` **must** emit `{ "summaries": [...], "failed": <int> }` with one summary per suite.
- [ ] Each summary **must** carry `suite`, `total`, `passed`, `failed`, and a `results` array of `{ id, name, suite, pass, detail }`.
- [ ] `chant bench --suite=retrieval` **must** run only retrieval scenarios; `--suite=e2e` only the e2e suite; `all` (default) both.
- [ ] When any scenario fails, `chant bench` **must** exit `1`; when all pass it **must** exit `0`.
- [ ] The retrieval suite **must** include at least the shipped scenarios `RET-001`..`RET-004` (hit, true-negative, refund routing, column disambiguation).
- [ ] The e2e suite **must** run each verifier-bearing recipe through run + verify and assert `verifier passed → trusted`; recipes with no verifier or no example **must** be reported as skipped passes.
