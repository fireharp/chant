# US-006 — Verify passing makes a reuse result trusted

**As an** agent reusing a recipe
**I want** `chant verify` to run the procedure then the verifier and bless the result only on a pass
**So that** I trust the output only when it actually worked here.

## Acceptance criteria

- [ ] Running `chant verify --json csv-revenue-by-channel` **must** exit `0` when the verifier passes.
- [ ] The outcome **must** set `"subcommand": "verify"`, `"verifier_ran": true`, `"exit_code": 0`, and `"trusted": true`.
- [ ] The outcome `message` **must** be `verifier passed — result is trusted`.
- [ ] By default (`--run` true) verify **must** run the procedure before the verifier; `--run=false` **must** skip the procedure and verify only.
- [ ] A passing verify **must** record a run on the recipe (`metrics.runs` increments, `metrics.last_success_at` set) so the track record feeds future retrieval.
- [ ] When the recipe ships an `examples[0].input`, verify **must** auto-fill `{{input}}` / `{{input_file}}` from it if not provided via `--input`.
- [ ] NOTE: `--json`, `--input`, and `--run` **must** precede the recipe id on the command line.
