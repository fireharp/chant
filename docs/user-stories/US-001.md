# US-001 — Suggest finds a reusable recipe before writing new code (hit)

**As an** agent about to write code for a recurring task
**I want** `chant suggest` to surface a matching verified recipe as a candidate
**So that** I reuse the tested method instead of re-solving from scratch.

## Acceptance criteria

- [ ] Running `chant suggest --task "compute revenue by channel from this csv" --files "orders.csv" --columns "utm_source,amount" --json` **must** exit `0` and emit an outcome with `"subcommand": "suggest"`.
- [ ] The outcome **must** set `"match_found": true` and contain a non-empty `hits` array.
- [ ] The top hit **must** have `"id": "csv-revenue-by-channel"` and `"verifier_exists": true`.
- [ ] Each hit **must** carry a `confidence` (0–1, rounded to 2 dp) and a `reuse_command` of the form `chant verify <id>   # run + verify before trusting`.
- [ ] The outcome **must** set `recommended_next_command` to the top hit's `reuse_command` (never to a "trust now" instruction).
- [ ] A hit **must not** set `trusted: true`; retrieval ranks candidates, it does not bless them.
- [ ] Hits **should** include human-readable `reasons` (e.g. "task text overlaps recipe description/patterns", "input files match recipe file signal", "input columns satisfy recipe column aliases").
