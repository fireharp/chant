# US-018 — AI data analyst reuses a CSV revenue enchantment across Shopify/Stripe/custom exports

**As an** AI data analyst handed a new messy orders export
**I want** chant to retrieve and verify the `csv-revenue-by-channel` enchantment instead of re-deriving the analysis
**So that** I compute revenue-by-channel deterministically and cheaply, robust to column-name drift.

This is canonical Demo 1 (CSV revenue). The enchantment's `columns_any` alias
groups (`channel|source|utm_source`, `revenue|amount|price|total`) absorb schema
drift between Shopify, Stripe, and custom exports.

## Acceptance criteria

- [ ] Running `chant suggest --task "compute revenue by channel" --files "orders_shopify.csv" --columns "utm_source,amount" --json` **must** return `csv-revenue-by-channel` as the top hit with `verifier_exists: true`.
- [ ] Running the same suggest with a Stripe-flavored schema (`--columns "source,price"`) **must** still return `csv-revenue-by-channel` as the top hit (column aliases cover the drift).
- [ ] The hit `reasons` **should** include `input columns satisfy recipe column aliases` when the available columns cover all required alias groups.
- [ ] Running `chant verify --json csv-revenue-by-channel` **must** execute `run.py` then `test_recipe.py`, produce the `revenue_by_channel.json` artifact, and set `"trusted": true`.
- [ ] A query whose columns satisfy none of the alias groups (e.g. `--columns "foo,bar"`) **must not** earn the column-signal contribution (the file-glob and lexical signals may still match).
- [ ] The reuse path **must** be verifier-first: a suggest hit alone **must** carry `trusted: false`; trust appears only in the `verify` outcome.
