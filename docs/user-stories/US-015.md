# US-015 — Doctor validates config and store health

**As a** maintainer setting up or debugging chant
**I want** `chant doctor` to validate config, recipes dir, verifier coverage, and gitignore
**So that** I catch misconfiguration before it bites.

## Acceptance criteria

- [ ] Running `chant doctor --json` **must** emit `{ "checks": [...], "ok": <bool> }` where each check has `name`, `status` (`ok`|`warn`|`fail`), and `detail`.
- [ ] Doctor **must** include checks named `config`, `recipes-dir`, `verifiers`, and `gitignore`.
- [ ] The `verifiers` check **must** be `warn` when any recipe lacks a verifier, reporting `<N>/<M> recipe(s) have no verifier — reuse can't be trusted`, and `ok` otherwise.
- [ ] When `chant.yml` is absent the `config` check **must** be `warn` advising `run \`chant init\``.
- [ ] When the recipe library fails to load, the `recipes` check **must** be `fail` and `chant doctor` **must** exit `1`.
- [ ] When there are no `fail` checks, `chant doctor` **must** exit `0` and (human mode) print `doctor: no blocking issues.`
